What's the point?
=================
BASSALAC is an extension to the BASS audio library, enabling the playing of
ALAC (Apple Lossless Audio Codec) encoded files.

NOTE: The ALAC format is supported as standard by BASS via the OS's codecs on
      macOS and iOS, and also on Windows 10.


Requirements
============
BASS 2.4 is required.


Using BASSALAC
==============
The plugin system (see BASS_PluginLoad) can be used to add ALAC support to the
standard BASS stream (and sample) creation functions. Dedicated ALAC stream
creation functions are also provided by BASSALAC.

The usage information in the BASS.TXT file (from the BASS package) is also
applicable to BASSALAC and other add-ons.

TIP: The BASSALAC.CHM file should be put in the same directory as the BASS.CHM
     file, so that the BASSALAC documentation can be accessed from within the
     BASS documentation.

History
=======
These are the major (and not so major) changes in each release, along with the
functions/options that are introduced/affected by them. There are other little
tweaks and fixes made along the way too.

2.4.1 - 31/7/2024
-----------------
* Support for 64-bit MP4 atoms (very large files)
	BASS_ALAC_StreamCreateFile/User/URL
* Error code for unstreamable atom order changed to match BASS
    BASS_ERROR_UNSTREAMABLE (BASS_ALAC_StreamCreateURL error code)
    BASS_ERROR_MP4_NOSTREAM removed
* MP4 tag reading removed and handled by BASS instead on all platforms
    BASS_TAG_MP4 (BASS_ChannelGetTags type)
* Channel order fix for 3/5/7 channel files

2.4.0.2 - 6/1/2021
------------------
* Fix for mono 24-bit ALAC files
* Support for Description/Keywords/Category/Grouping/Lyrics tags on Linux (already on other platforms)

2.4 - 2/2/2016
--------------
* First release
